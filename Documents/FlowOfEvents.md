# Потоки событий
## Поток событий для прецедента «Создать группу»
Основной поток:
1. Вариант использования начинается, когда пользователь отправляет боту команду /creategroup.
2. Бот запрашивает название для создаваемой группы.
3. Происходит проверка на существование группы с заданным названием.
   Если такой группы не существует переход пункту 4.Иначе выполняется альтернативный поток А1.
4. Бот запрашивает пароль для группы.
5. Група создается.Пользователю отправляется сообщение о успешном создании группы.
5. Вариант использования завершается.

Альтернативный поток событий А1:
1.Сообщение о существовании группы с таким именем.
2.Вариант использования завершается.

## Поток событий для прецедента «Вступить в группу»
Основной поток:
1. Вариант использования начинается, когда пользователь отправляет команду /join.
2. Бот запрашивает название группы.
3. Происходит проверка на существование группы с заданным названием.
   Если такая группа существует переход к пункту 4.Иначе выполняется альтернативный поток А1.
4. Бот запрашивает пароль.
5. Происходит проверка пароля.Если пароль верный переход к пунку 6.
   Иначе выполняется альтернативный поток А2.
6. Пользователь добавляется в группу.
7. Пользователь получает сообщение об успешном вступлении в группу.
8. Вариант использования завершается.

Альтернативный поток событий А1:
1. Сообщение о отсутствии группы с таким именем.
2. Вариант использования завершается.

Альтернативный поток событий А2:
1. Сообщение о неверно вееденном пароле.
2. Проверка количества попыток введения пароля.
   Если количество попыток ввода пароля превысило 3 переход к пункту 3 А2
   Иначе переход к пункту 4 основного потока
3.Вариант использования завершается.

## Поток событий для прецедента «Узнать человека для подарка»
Основной поток:
1. Вариант использования начинается, когда пользователь отправляет команду /get.
2. Бот запрашивает название группы.
3. Происходит проверка на существование группы с заданным названием.
   Если такая группа существует переход к пункту 4.Иначе выполняется альтернативный поток А1.
4. Бот запрашивает пароль.
5. Происходит проверка пароля.Если пароль верный переход к пунку 6.
   Иначе выполняется альтернативный поток А2.
6. Происходит случайный выбор одного из членов группы.
7. Выбраный пользователь удаляется из группы(для исключения возможности его повторного выбора).
8. Пользователь получает сообщение с случайно выбранным именем из списка данной группы.
9. Пользователь получает сообщение с симпатичной новогодней картиночкой.
10. Вариант использования завершается.

Альтернативный поток событий А1:
1. Сообщение о отсутствии группы с таким именем.
2. Вариант использования завершается.

Альтернативный поток событий А2:
1. Сообщение о неверно вееденном пароле.
2. Проверка количества попыток введения пароля.
   Если количество попыток ввода пароля превысило 3 переход к пункту 3 А2
   Иначе переход к пункту 4 основного потока.
3.Вариант использования завершается.
